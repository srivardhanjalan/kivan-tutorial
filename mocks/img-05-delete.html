DELETE /users/me — three decisions
# 1. Clerk FIRST — if it fails, nothing changed and the user retries
  DELETE api.clerk.com/v1/users/{id} → 200 (record-first would orphan live credentials)
# 2. flag the record — unconditional; never purge it, later steps point here
  update_item SET is_deleted = true
# 3. the guard rides on every MUTATING write, re-checked by DynamoDB
  PUT /users/me  ConditionExpression: …not is_deleted… → 403
# and the just-deleted session is dead — reads bounce too
  GET /users/me → 403   re-login → Couldn't find your account.
